# Contributing to Horizon-Tools

Thanks for your interest in contributing! This project is maintained by Michael Petschel under the umbrella of Michael Petschel Media. Contributions are welcome, whether that's a bug report, a feature suggestion, or a pull request.

This project is licensed under the [PolyForm Noncommercial License 1.0.0](./LICENSE). By submitting a contribution, you agree that:

- You have the right to submit the contribution (e.g. it's your own original work, or you have permission to share it).
- You understand the project — including your contribution — cannot be used commercially without permission from the project owner.

If that doesn't work for you, feel free to open an issue to discuss before submitting a PR.

## Reporting Bugs

Before opening a new issue, please search existing issues to avoid duplicates. When filing a bug report, include:

- A clear, descriptive title
- Steps to reproduce the issue
- What you expected to happen vs. what actually happened
- Environment details (OS, browser, version, etc. — whatever's relevant)
- Screenshots or error logs, if applicable

## Suggesting Features

Feature requests are welcome! Open an issue and include:

- The problem you're trying to solve
- Your proposed solution or approach
- Any alternatives you've considered

## Submitting a Pull Request

1. **Fork the repo** and create your branch from `main`.
2. **Name your branch** something descriptive, e.g. `fix/login-redirect` or `feature/dark-mode`.
3. **Make your changes.** Keep commits focused — one logical change per commit is easier to review than one giant commit.
4. **Write clear commit messages.** A short summary line, plus more detail in the body if needed.
5. **Test your changes** before submitting. If the project has tests, make sure they pass. If it doesn't yet, at least manually verify your change works as expected and doesn't break existing functionality.
6. **Open a pull request** against `main`. In the PR description, explain:
   - What the change does and why
   - Any related issue numbers (e.g. `Closes #12`)
   - Anything you're unsure about or want feedback on

## Code Style

- Match the existing style and structure of the file you're editing.
- Keep functions and components focused — small, readable pieces over large, tangled ones.
- Comment non-obvious logic, but don't over-comment the obvious.

## Review Process

Pull requests are reviewed by the maintainer. You may be asked to make changes before a PR is merged — this is normal and not a reflection on the quality of your idea, just part of keeping the codebase consistent. Response times may vary since this is a personal project maintained outside of full-time work.

## Questions?

If anything here is unclear, or you want to discuss an idea before building it out, open an issue and tag it `question` or `discussion`.

Thanks again for helping improve Horizon-Tools!

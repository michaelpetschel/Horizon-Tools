# Security Policy

## Supported Versions

Horizon-Tools is currently maintained on the `main` branch. Security fixes are applied there and released as soon as practical.

## Reporting a Vulnerability

If you discover a security vulnerability in Horizon-Tools, please **do not open a public issue**. Instead, report it privately so it can be addressed before details are public.

**Preferred method:** Use GitHub's [private vulnerability reporting](https://github.com/michaelpetschel/Horizon-Tools/security/advisories/new) feature (Security tab → "Report a vulnerability").

**Alternative:** Open a blank GitHub issue titled `[Security] Contact request` with no details, and a way to reach you privately will be arranged.

When reporting, please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof-of-concept
- Affected version/commit
- Any suggested fix or mitigation, if you have one

## What to Expect

- **Acknowledgment:** within a few days of your report.
- **Assessment:** the issue will be investigated and you'll be updated on next steps.
- **Fix & disclosure:** once a fix is ready, it will be released and the reporter credited (if desired) in the release notes or advisory. Coordinated disclosure is appreciated — please allow time for a fix before public disclosure.

## Scope

This policy covers the code in this repository. It does not cover third-party dependencies, services this project may integrate with, or issues arising from self-hosted/modified deployments unrelated to the upstream code.

## Recognition

Responsible disclosure is genuinely appreciated. Reporters who follow this process will be credited in the fix's release notes unless they prefer to remain anonymous.
